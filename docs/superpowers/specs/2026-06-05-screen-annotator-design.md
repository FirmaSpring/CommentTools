# 桌面批注工具 (Screen Annotator) 设计规范

## 1. 项目概述
本项目是一个基于 Electron 框架的跨平台桌面批注工具。旨在提供一个轻量、现代、且不干扰用户正常操作的屏幕绘制工具。用户可以在桌面的任何位置（其他软件、壁纸之上）进行高自由度的批注。

## 2. 核心交互与视觉设计
### 2.1 悬浮小黑球 (Floating Widget)
- **静默状态**：在屏幕右侧放置一个小巧的黑色球体。
- **悬停效果**：当鼠标移动到黑球上时，黑球放大以提供视觉反馈。
- **展开交互**：点击黑球后，展开主批注工具栏。

### 2.2 主工具栏 (Main Toolbar)
- **布局**：垂直展开（从上到下排列），采用磨砂玻璃质感（Frosted Glass），可以透出底层背景。
- **包含功能**：
  1. 鼠标（指针穿透，允许操作底层应用）
  2. 笔（进行批注）
  3. 橡皮（擦除批注）
  4. 撤回（Undo）
  5. 清屏（Clear All）
  6. 收起（折叠工具栏）
- **UI 细节**：所有图标使用 SVG 格式，带有小幅度的悬停/点击动画。
- **状态反馈**：点击某个功能图标后，该图标高亮，同时鼠标在绘图区域变为对应的图标样式（在工具栏内部仍保持默认箭头样式）。

### 2.3 二级设置面板 (Submenu Panel)
- **触发方式**：再次点击已高亮的工具图标（如“笔”）。
- **展开效果**：侧边滑出（Side Popout），配合 0.3 秒的“水波扩散（Ripple）”高级过渡动画。
- **包含设置（以“笔”为例）**：
  - 颜色选择器
  - 书写大小（粗细）滑块
  - 滑动轨迹长度（拖尾效果，默认为 0，可自定义轨迹颜色）

## 3. 技术架构设计

### 3.1 窗口管理 (Electron BrowserWindow)
应用包含两个主要的隐藏窗口层：
1. **控制层窗口 (Toolbar Window)**：
   - 负责承载小黑球、主工具栏和二级设置面板。
   - 永远置顶 (`alwaysOnTop: true`)。
   - 具有透明背景 (`transparent: true`) 且无边框 (`frame: false`)。
2. **绘制层窗口 (Canvas Window)**：
   - 全屏覆盖所有显示器。
   - 背景完全透明 (`transparent: true`)。
   - 永远置顶 (`alwaysOnTop: true`)，层级在控制层窗口之下。
   - 鼠标事件穿透管理：当选择“鼠标”工具时，设置 `setIgnoreMouseEvents(true)` 让事件穿透到下层应用；当选择“笔/橡皮”时，设置 `setIgnoreMouseEvents(false)` 拦截事件进行绘制。

### 3.2 绘制技术方案 (HTML Canvas)
使用 HTML `<canvas>` 作为核心绘制引擎。
- **原因**：性能卓越，且非常适合实现高级动画特效（如要求中的拖尾/滑动轨迹效果）。
- **线条平滑**：使用贝塞尔曲线算法（Quadratic Bezier）连接鼠标移动的采样点，确保笔触平滑。
- **拖尾效果实现**：通过保留绘制历史队列，并在每帧渲染时在旧路径上应用带有透明度的覆盖层，或动态计算带有衰减特性的自定义路径来实现。
- **撤销/重做**：保存绘制动作栈，撤销时清空画布并重新播放栈内历史操作。

### 3.3 状态管理
由于有两个独立的渲染进程（工具栏和画布），需要使用 Electron 的 IPC (Inter-Process Communication) 进行状态同步。
- 当工具栏切换工具或修改颜色/粗细时，通过 IPC 通知画布窗口更新绘制上下文 (Context) 状态。
