# 桌面批注工具退出按钮与画笔图标设计

## 1. 目标
- 在现有“系统设置”面板中新增一个 `退出程序` 按钮。
- 点击该按钮后，完整退出当前 Electron 应用，而不是只隐藏某个窗口。
- 为程序新增一套“画笔”主题图标，用作应用图标与安装器图标来源。

## 2. 设计范围
- 不新增主工具栏按钮位。
- 不改变当前 `settings` 面板的入口方式。
- 不修改现有快捷键、自启动、OCR、辅助模式等已有功能行为。

## 3. 退出程序按钮设计

### 3.1 位置
- 放在 `系统设置` 面板底部。
- 位于“快捷键设置”区域之后，作为一个独立动作区。

### 3.2 视觉
- 使用现有危险操作按钮风格体系。
- 文案固定为 `退出程序`。
- 颜色沿用当前危险操作红色语义，但不额外引入更重的确认弹窗，保持操作直接。

### 3.3 行为
- 点击按钮后，渲染进程通过 IPC 请求主进程退出应用。
- 主进程收到请求后执行 `app.quit()`。
- 退出逻辑应复用 Electron 正常退出流程，确保全局快捷键注销与窗口清理仍走既有 `will-quit` 流程。

## 4. 画笔图标设计

### 4.1 风格
- 图标主体为简洁画笔造型。
- 风格与现有工具栏一致，偏克制、干净、可读。
- 优先采用深色轮廓 + 浅色主体方案，保证在浅色和深色背景下都可辨识。

### 4.2 资源形式
- 先生成一份 SVG 源图。
- 再输出可供 Electron 使用的 PNG / ICO 打包资源。
- 图标资源放入项目内固定资源目录，避免散落。

### 4.3 使用位置
- 开发态窗口图标。
- 打包后的应用图标。
- NSIS 安装器图标。

## 5. 架构与职责

### 5.1 渲染层
- `SystemSettingsPanel.tsx`
  - 新增 `退出程序` 按钮。
  - 暴露 `onQuitApp` 回调。

### 5.2 主进程
- `electron/main.ts`
  - 新增 `app:quit` IPC 处理。
  - 调用 `app.quit()` 退出程序。

### 5.3 打包配置
- `package.json`
  - 在 `build` 配置中接入应用图标路径。
  - 让 Windows 打包和 NSIS 安装器使用画笔图标。

## 6. 测试策略
- 单元测试：
  - 锁住系统设置面板包含 `退出程序` 按钮文案。
  - 锁住主进程存在 `app:quit` IPC 处理逻辑。
  - 锁住打包配置包含图标路径。
- 构建验证：
  - `npm test`
  - `npm run lint`
  - `npm run build`
  - `npm run dist:win`

## 7. 风险与控制
- 风险：退出按钮误放到主工具栏导致误触。
  - 控制：仅放在系统设置面板底部。
- 风险：只关闭窗口而没有真正退出进程。
  - 控制：主进程统一执行 `app.quit()`。
- 风险：安装器仍使用默认 Electron 图标。
  - 控制：在打包配置中显式声明图标资源路径，并跑完整安装器构建验证。

## 8. 验收标准
- 系统设置面板底部出现 `退出程序` 按钮。
- 点击后程序完整退出。
- 应用和安装器图标替换为画笔图标。
- 测试、lint、build、Windows 安装器构建通过。
